## 🎥 演示 1：检查软件是否安装

### 场景：打开 PowerShell

**第1步 - 打开菜单**
```
看着屏幕，找到键盘上的 Windows 键（有个Windows标志的）
然后同时按下 R 键
```

**你会看到**：
```
┌─────────────────┐
│ 运行            │
├─────────────────┤
│ 输入命令：      │
│ powershell      │ ← 在这里输入
└─────────────────┘
```

**第2步 - 输入命令**
```
用鼠标点进输入框，然后打字：
powershell

打完后按 Enter 键
```

**你会看到**：
```
Windows PowerShell
版权所有 (C) Microsoft Corporation。
保留所有权利。

PS C:\Users\Administrator>  ← 光标在这里闪烁
```

✅ **好的！PowerShell 打开了！**

---

### 第3步 - 检查 Node.js

**在 PowerShell 中输入**：
```powershell
node -v
```

**然后按 Enter 键**

**可能的结果**：

❌ **如果看到这样**：
```
'node' 不是内部或外部命令，也不是可运行的程序或批处理文件。
```
→ Node.js 没装，需要安装

✅ **如果看到这样**：
```
v20.10.0
```
→ 装好了，版本号是 v20.10.0

---

### 第4步 - 检查 Python

**继续在 PowerShell 中输入**：
```powershell
python --version
```

**按 Enter 键**

**可能的结果**：

❌ **如果看到这样**：
```
'python' 不是内部或外部命令...
```
→ Python 没装

✅ **如果看到这样**：
```
Python 3.11.0
```
→ 装好了

## 🎯 常见场景处理

### 场景1：我想查看计算器有没有更新

**第1步 - 打开项目文件夹**
```powershell
start .
```

**第2步 - 进入代码目录**
```
src → data → calculator.ts
```

**第3步 - 打开 calculator.ts（用记事本）**
```
找这一行：
export const DEFAULT_FUEL_SURCHARGE = 0.18;

0.18 = 18%（被更新的燃油费）
```

✅ **看到新数字就说明爬虫工作成功了！**

---

### 场景2：爬虫出错了

**第1步 - 看错误信息**
```
PowerShell 中会显示红色的错误信息
复制这个错误信息
```

**第2步 - 查看日志文件**
```
打开文件夹：logs
找到文件：scraper_log.txt
用记事本打开，看详细的出错信息
```

**第3步 - 对号入座**
```
错误信息中有 "Module not found"
  → 重新安装：pip install playwright requests beautifulsoup4

错误信息中有 "Timeout"
  → 网络问题，等一会儿再试

错误信息中有 "Element not found"
  → DHL官网改版了，告诉我
```

---

### 场景3：我想手动修改燃油费

**第1步 - 打开代码文件**
```
用任意文本编辑器打开：
src/data/calculator.ts
```

**第2步 - 找到这一行**
```typescript
export const DEFAULT_FUEL_SURCHARGE = 0.18;
```

**第3步 - 修改数字**
```
0.18 改成 0.20  → 20% 燃油费
0.18 改成 0.15  → 15% 燃油费

改完后按 Ctrl+S 保存
```

**第4步 - 项目会自动刷新**
```
如果项目在运行（npm run dev），
会自动重新加载计算器
```

✅ **完成！燃油费已修改！**

---

### 场景4：我想停止自动运行

**说白了就是：禁用定时任务**

1. 进入 WorkBuddy 的自动化管理
2. 找到 "DHL燃油费自动更新爬虫"
3. 改成"暂停"状态

✅ **以后爬虫就不会自动运行了**

如果想重新启用，就改回"启用"状态。
